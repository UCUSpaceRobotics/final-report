The rover uses a dedicated hardware safety system, including power switching and emergency shutdown, independent of the main control electronics (\autoref{fig:hardware_safety}). An INA-based module (D1) monitors battery voltage and current for state estimation and fault detection.

Rover status is indicated by a 360° industrial signal lamp mounted at its highest point. Four colors represent the operating state: orange (power on), green (manual mode), red (autonomous mode), and blue (wireless connection established).

The electronics enclosure is sealed with dedicated covers for removable modules, reducing dust and water ingress while ensuring reliable outdoor operation.

The Emergency Stop (E-Stop) system is responsible for immediately and
unconditionally removing power from the rover's actuation systems,
independent of the state of the onboard software. Two independent
trigger paths converge on the same power-cutting hardware, described in
Section~\ref{sec:estop-hw}.

\subsubsection{Reliability Argument}
\label{sec:estop-reliability}

The emergency stop design provides reliability through the following
properties:

\begin{itemize}
    \item \textbf{Hardware independence:} the power-cutting path
    (relay/transistor stage) does not depend on the main onboard
    computer or any software being operational. A full software
    crash or main-computer failure does not prevent the E-Stop from
    functioning.
    \item \textbf{Dual, independent trigger paths:} both the physical
    button and the dedicated emergency RF channel are able to trigger
    the same power-cutting hardware independently of one another,
    providing redundancy — failure of one path (e.g. RF interference)
    does not disable the other.
    \item \textbf{Dedicated communication channel:} the remote trigger
    path uses a frequency/channel reserved exclusively for emergency
    signalling, decoupled from the nominal control/telemetry link, so
    it remains available even if the primary link is degraded or lost.
