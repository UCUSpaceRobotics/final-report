The rover uses a dedicated hardware safety system, including power switching and emergency shutdown, independent of the main control electronics (\autoref{fig:hardware_safety}). An INA-based module (D1) monitors battery voltage and current for state estimation and fault detection. The switching relay is normally-closed, so any loss of control signal also defaults to the safe, power-off state.

Rover status is indicated by a 360\textdegree{} industrial signal lamp mounted at its highest point. Four colors represent the operating state: orange (power on), green (manual mode), red (autonomous mode), and blue (wireless connection established).

The electronics enclosure is sealed with dedicated covers for removable modules, reducing dust and water ingress while ensuring reliable outdoor operation.

The Emergency Stop (E-Stop) system removes power from the rover's actuation systems immediately and unconditionally, independent of onboard software state. Two independent trigger paths converge on the power-cutting hardware shown above.

\subsubsection{Reliability Argument}
\label{sec:estop-reliability}

The design is reliable because the power-cutting path is purely hardware and does not depend on the main computer or any software, both trigger paths (button and dedicated RF channel) act on the same relay independently of each other, and the emergency RF channel is separate from the nominal control link so it stays available if that link is jammed or lost.

The E-Stop has been triggered repeatedly during testing (via both the physical button and the emergency RF channel) and has activated correctly every time.

Test evidence for each trigger path is reported together with the overall test campaign in \autoref{sec:test_plan}
